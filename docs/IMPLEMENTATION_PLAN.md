## Phase 1 — Data Pipeline (MUST COMPLETE BEFORE POWER BI)

**Script:** `prepare_data.py`

### Steps in order:
1. **Step 1 — Load & Audit** — Print column names, dtypes, null counts, shape for all 3 source files
2. **Step 2 — Build `fact_claims`** — Map fields, calculate `days_to_close`, synthesize `fraud_flag` & `catastrophe_flag`
3. **Step 3 — Build `fact_premiums`** — One row per policy per active year; synthesize GWP from LOB ranges where needed
4. **Step 4 — Build Dimensions** — `dim_policy`, `dim_customer`, `dim_geography` (hardcoded 50 states), `dim_date` (2020-2025)
5. **Step 5 — Referential Integrity Check** — Assert all FK values in fact tables exist in dim tables; print PASS/FAIL table
6. **Step 6 — Write Outputs** — Write 6 CSVs to `/data/clean/` with UTF-8 encoding

### Gate Criteria (all must pass before Phase 2):
- [ ] Exit code 0
- [ ] 6 files present in `/data/clean/`
- [ ] `fact_claims`: 8,000–12,000 rows
- [ ] `fact_premiums`: 10,000–18,000 rows
- [ ] `dim_date`: exactly 2,192 rows (2020-01-01 to 2025-12-31)
- [ ] `dim_geography`: exactly 50 rows
- [ ] No nulls in any FK column
- [ ] Referential integrity check: all 6 assertions PASS

## Phase 2 — Power BI Foundation

### 2.1 Import Order
1. Open Power BI Desktop (fresh file)
2. **Apply theme first**: View → Themes → Browse → `apex_insurance_theme.json`
3. Get Data → Text/CSV → import all 6 files from `/data/clean/`
4. In Power Query: set all types **explicitly** (no auto-detect)

### 2.2 Type Mappings in Power Query
| Column Pattern | PQ Type |
|---------------|---------|
| `*_date`, `date` | Date |
| `*_flag`, `is_*`, `renewal_flag` | True/False |
| `date_key` | Whole Number |
| `gross_written_premium`, `*_loss`, `*_score` | Decimal Number |
| All IDs, codes, names | Text |

### 2.3 Relationships (Model View)
Configure all 6 relationships as **Single direction** (fact → dim):

| From (Many) | To (One) | Key |
|-------------|----------|-----|
| `fact_claims` | `dim_date` | `date_key` |
| `fact_claims` | `dim_policy` | `policy_id` |
| `fact_claims` | `dim_customer` | `customer_id` |
| `fact_claims` | `dim_geography` | `geography_id` |
| `fact_premiums` | `dim_date` | `date_key` |
| `fact_premiums` | `dim_policy` | `policy_id` |

> **Zero bidirectional relationships. Enforce this before creating any measures.**
